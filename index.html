Prework Study Guide

HTML

  • The!DOCTYPE declaration tells the browser what document type it should expect when it is opened.
  • The html opening tag represents the root, or start, of the HTML document.
    • The lang=... attribute tells the browser the language for the webpage.
  • The head element contains information about the webpage.
    • The title element defines the title for a webpage and it's important for Search Engine Optimization (SEO).
    • The style element defines the CSS styles associated with a website.
    • The base element defines the base URL for a webpage.
    • The link element connects an external resource to the HTML document.
    • The meta element defines e.g. the character set, description, keywords, author, and viewport.
      • The charset element sets our character encoding.
      • The http-equiv element sets an HTTP header for content.
      • The viewport element sets the default view for our webpage.
    • The script element embeds client-side scripts in the HTML document.
  • The body element represents the visible content shown to the user.
    • The header element appears at the top of the page and usually contains a navigation menu and/or introductory content.
    • section elements are semantic elements that define a section in a document that contains thematic content.
    • The id attribute identifies elements and selects them.
    • The div element creates a division in the page.
    • The span element creates an inline division.
    • The h1, h2, h3, h4, h5, and h6 elements represent the level of heading a given text block represents.
    • The p element represents a paragraph or block of text.
    • The ul, ol, and li elements represent unordered lists, ordered lists, and list items.
    • The img element contains information about images that are displayed on the webpage.
      • The src attribute defines the location of the image.
    • The br element creates a line of empty space, or a line break, between two blocks of content.
    • The a element (which stands for anchor) in a hyperlink creates links to the same webpage or other webpages.
    • The href attribute points to the URL for an hyperlink.
    • The footer element appears at the bottom of the page and usually contains the author, copyright, contact, sitemap, and navigation.

CSS

  • Inline styles apply to a specific HTML tag, using a style attribute with a CSS rule to style a particular page element only.
  • When using an external style sheet, we link the file using a link element placed in the HTML's head.
  • A margin indicates how much space we want around the outside of an element.
  • A padding indicates how much space we want around the content inside an element.
  • A selector defines the element or attributes to which the rules, or declarations, will apply.
  • Declarations contain two important components: the CSS property we want to apply and the value of the property.
  • Sometimes, we want to apply a rule to all the elements visible on the page. To do that, we use a special wild card symbol, or *
  • Do Not Repeat Yourself (DRY) is a coding principle that encourages developers to reuse or share pieces of code in order to reduce the number of lines of code that need to be written or maintained. DRY can greatly reduce the overall size and complexity of a codebase.
  • Besides text, we can also apply CSS rules to images.
  • The block value assigns each image to start on a new line.

Git

  • The ls command allows us to see a list of files and directories on our computer.
  • The cd command allows us to change directories.
  • The pwd command will print out the directory that we are currently in to the terminal.
  • The cd .. command will allow us to navigate backwards one directory at a time.
  • The mkdir command allows us to create a new directory.
  • The touch command allows us to create a new file.
  • The ssh-agent is a program that keeps track of a user's identity keys and passphrases.
  • The git revert command allows us to go back to earlier versions of our code if we need to.
  • The code . command opens VS Code.
  • The main is the default branch-name.
  • The git command must precede every Git statement.
  • The checkout command is to move the working branch to a new branch.
  • The -b flag creates a new branch.
  • The pre-fix feature indicates that a new branch is a feature.
  • The add command adds modifications in the current working branch to the staging area.
  • The -A flag indicates that we want to add all changes.
  • The git commit command needs to be entered with a commit message that describes what the commit contains.
  • The pull command receives a branch's modifications into the local environment.
  • The push command pushes the changes we have locally to our remote GitHub branch.
  • The -m flag associates a message with our commit.
  • The git status command checks what branch we are currently on.
  • The git checkout -b branch-name command creates a new branch and switches to it.
